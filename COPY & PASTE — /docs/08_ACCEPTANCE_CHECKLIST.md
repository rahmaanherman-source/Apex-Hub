# ACCEPTANCE CHECKLIST
# Definition of Done

## Purpose

This document defines when a feature, module, or service is considered “done” in APEX Hub.

---

## Checklist

A feature is **accepted** when:

- [ ] Complies with `00_CONSTITUTION.md`.  
- [ ] Aligns with `01_ENGINEERING_STANDARD.md`.  
- [ ] Uses `02_DESIGN_SYSTEM.md` components and patterns.  
- [ ] Respects `03_AI_BUILDER_RULES.md`.  
- [ ] Fits within `04_ARCHITECTURE.md`.  
- [ ] Has a complete spec per `05_FEATURE_SPECIFICATION.md`.  
- [ ] Deploys according to `06_DEPLOYMENT_STANDARD.md`.  
- [ ] Meets `07_SECURITY_STANDARD.md`.  
- [ ] Has tests for core behavior.  
- [ ] Emits analytics.  
- [ ] Integrates with Automation (if applicable).  
- [ ] Registers objects in Global Object Registry.  
- [ ] Has documented rollback or undo path.  

Only then is it considered part of APEX Hub.  
Anything less is not done.
